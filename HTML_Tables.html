What are the advantages of using HTML table tags?

HTML table tags are used to properly display and organize tabular data that is best explained visually.

These tags also make tabular data accessible to visually impaired users. Screen readers recognize tabular data when encapsulated in HTML table tags and will express such information appropriately to their visually impaired users.

What are the disadvantages?

HTML table tags can quickly clutter the code making it difficult to read.

They are meant only for tabular data and cannot be used for other types of data.

What are the HTML table tags?

  • <table> : presents tabular data in a 2D table of rows and columns
  • <thead> : surrounds of group of column headers
  • <tbody> : surrounds a set of table rows that present the table's body
  • <tfoot> : surounds the row that summarizes the tabular data, usually the total or another calculation. Will automatically display this row as the last row in the table
  • <tr> : defines a table row by surrounding a group of table cells
  • <th> : header cells, can be used for either column or row and usually accompanied by the appropriate attribute
  • <td> : table cells, usually only for the table's body
  • <colgroup> : surrounds all <col> tags to set a particular style to the specified column. This appears within the <table> tag at the top before the actual rows and columns
  • <col> : encapsulates the style for it's associated column
  • <caption> : nestled at the top and within the <table> tag and summarizes the type of information displayed in the table

What are some useful attributes for these tags?

  • colspan="#" : spans across the specified column, unitless, used with <th>
  • rowspan="#" : spans across the specified row, unitless, used with <th>
  • span="#" : spans the styling across the specified number of columns when used with <col>
  • scope="col | row" : informs screen readers whether the cells are associated with the row or column headers, used with <th>
  • class="_" : targets the element for CSS or JavaScript
  • id= "_" : targets the element for CSS or JavaScript and for association with other cells when applied to <th>
  • headers="_" : assoicates the cell with certain headers, used with <td>

What are nesting tables?

Tables within tables. Generally they are not advisable because they are not as easily read as a table that just had more cells for all that data.

The only time you would want to use nesting tables is if you needed to import a table from another source.

This is what a nested table looks like:

title1 title2 title3
cell1 cell2 cell3
cell2 cell3
cell4 cell5 cell6

Why and how should we associate cells with headers?

Not only is it a good for organizing your HTML document, associating cells and headers allows visually impaired users to understand the tabular data and how different pieces of information relate to each other.

There are two ways to do this and both use different attributes.

Method 1: "Scope" Attribute

Probably the easier of the two, simply include the scope attribute to all header cells (<th>) and specify whether it's a column header (scope="col") or row header (scope="row").

Method 2: "id" & "headers" Combination

A bit more complicated, but this method allows you to associate body cells (<td>) to specific rows and columns by giving each header cell a unique id and each body cell the appropriate header, which should basically be the same as the related header's id.
