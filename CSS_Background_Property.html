CSS Background Property

What is the CSS Background Property?

The background property falls under the awesome category of shorthand properties.

The background property can specify up toeight background-related CSS properties at once:

  • background-attachement
  • background-clip
  • background-color
  • background-image
  • background-origin
  • background-position
  • background-repeat
  • background-size

When entering values for the background CSS property, the order does not matter, but one recommended order is:

[image], [position / size], [repeat], [attachment], [origin], [clip], [color]

Regardless of the exact order, the size value always comes right after the position value and must be separated with "/".

Also note that not all properties need to be specified to use the shorthand property; any property that isn't specified will be set to its default setting.

What are the background default settings?
Background Property Default Value
background-image none
background-position 0% 0%
background-size auto
background-repeat repeat
background-attachment scroll
background-origin padding-box
background-clip border-box
background-color transparent

In other words, specifiying:

  • "background: none",
  • "background: transparent", and
  • "background: repeat scroll 0% 0% / auto padding-box border-box none transparent"

all do the same thing and all are just the default setting.

The background is made of layers?

One of the coolest things about all background-related CSS properties is that they can specify multiple background layers. Each background-related property essentially represents a layer, but can be more than just one layer. These layers lay ontop of each other to determine how the background ultimately displays on screen.

When a background-related CSS property is made up of multiple layers, each layer is represented by a value that is separated from the other values by a comma. The first value in this list is the top layer. Regardless of how many layers there are, if color is specified, it will always be the last layer of the background.

Can the background be an image?

Background-image in particular can use more than one image.

In fact, a "background image" doesn't have to be a picture (i.e. png, jpeg, svg files) at all; a background image can be a gradient (More on that later).

When using actual pictures as background images, the file name of the picture has to be encapsulated within an url(). Without the url(), the computer won't know where to find the image.

This method requires an HTTP request, which might make the web page load slower. If that's an issue, a data URI can be used instead because it saves HTTP Requests. The data URI replaces the image file, so you still need to remember to encapsulate the data URI within the url(). You can use online conversion tools to get a data URI for your pictures.

Multiple background images are separated by commas. If using actual pictures, each picture has its own url(), and each url() is separated by commas.

If using gradients, the type of gradient must be specified.

  • conic-gradient()
  • linear-gradient()
  • radial-gradient()
  • repeating-conic-gradient()
  • repeating-linear-gradient()
  • repeating-radial-gradient()

Background-position dictates where the background images are displayed on screen.

Can the background change size?

Background-size controls the background images' sizes.

Can you repeat an image or pattern to use as a background?

Background-repeat specifies how to repeat background images.

By default, the background image is repeated from left to right, top to bottom.

Can the background "move"?

Background-attachment is similar to background-position, except instead of determining the background's image's exact location, it determines whether the image is fixed to where it's positioned, or if it scrolls along with the rest of the page when the user reads down the web page.

What is a positioning area?

Background-origin is another property related to position. It decides the positioning area of the background image(s).

What is a painting area?

Background-clip specifies the painting area of the background image.

Does the background have to be white?

Background-color literally colors the background.
